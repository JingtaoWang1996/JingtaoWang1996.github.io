# 基础定义

* 靶标：以CVE漏洞为例，网络靶场中的CVE靶标，本质就是**可复现特定CVE漏洞的标准化网络场景/设备环境**
  * 特点：可控且可复现【任何人使用同一个CVE靶标都能触发同样的效果】

# “太空靶场”-Space Range

节点仿真关键能力与约束

1. **拓扑弹性**：基于 SDI 按需弹性扩缩，单任务可快速部署 10–10,000 节点，支持节点克隆 / 回滚 / 快照，环境重建时间≤30 分钟。
2. **虚实结合**：兼容实物接入（如相控阵终端、星上载荷模拟器、地面网关），虚拟节点与实物节点通信时延≤5ms，数据一致性≥99.9%。
3. **流量仿真**：支持太空专属协议流量（如 CCSDS TM/TC）与地面网络流量混合模拟，单节点流量带宽 1Mbps–10Gbps，并发连接数≥100,000。
4. **安全闭环**：内置漏洞靶标库与攻击引擎，支持攻击路径可视化、攻击效果量化评估，提供 “攻击 - 检测 - 响应 - 恢复” 全流程验证。

## ACRE网络靶场

ACRE 是 ManTech 公司推出的虚实结合网络靶场（Cyber Range），核心是通过物理 + 虚拟化 + 软件定义基础设施（SDI）构建可控闭环环境，可完整复现实体与虚拟网络、系统及流量，用于攻防演练、安全测试、技术验证与人才培训，尤其适配卫星 C2、太空网络、关键信息基础设施等场景。以下是其核心定义与关键特征。

------

### 核心定义与内涵

ACRE 全称未公开，官方定位为 “先进测试、评估与网络培训平台”，核心是在安全可控的虚实融合环境中，复现真实 IT/OT/ 太空网络架构、用户行为、恶意流量与攻击场景，支持 “练为战” 的攻防闭环验证，同时保障合规留痕与环境可重置。

### 关键特征与核心能力

* **虚实结合架构**：兼容物理设备、虚拟机、容器与 SDI，支持实物接入与虚拟仿真混合部署，虚拟节点可弹性扩缩（单任务 10–10,000 节点），环境重建≤30 分钟。
* **全场景仿真**：覆盖 TCP/IP、CCSDS、5G NTN 等协议栈，支持星间链路、地面运控、工业控制、企业网等拓扑；可模拟链路损伤、协议异常、GPS 欺骗等 20 + 类攻击场景。

* **安全闭环能力**：内置漏洞靶标库、攻击引擎与防御验证模块，支持红队渗透、蓝队防御、应急响应演练与攻防效果量化评估，适配军工与商业级安全测试。

* **服务化交付**：提供 ReadyRange 即服务、Express 自动化定制与分层订阅，支持持久化访问与快速环境重置，适配个人测试、团队演练与大规模演习。

与传统靶场的差异

| 对比维度 | ACRE 网络靶场                             | 传统网络靶场                           |
| -------- | ----------------------------------------- | -------------------------------------- |
| 架构     | 虚实深度融合 + SDI 弹性调度               | 多以纯虚拟或纯物理为主，弹性差         |
| 场景适配 | 聚焦太空 C2、卫星网络、关键基础设施       | 以企业网、数据中心为主，太空场景适配弱 |

# 10w+节点的靶场的搭建途径

搭建**10 万 + 节点**的大规模网络靶场，核心是**依托虚拟化 / 容器化技术 + 分布式架构**，结合网络仿真与资源调度平台，实现节点的高密度部署、动态管理和隔离运行。这类靶场主要用于模拟超大规模网络拓扑（如城域骨干网、物联网集群、卫星通信网络），支撑攻防演练、协议验证、态势推演等需求。以下是主流搭建途径及关键技术方案：

### 基于**虚拟化技术**的搭建途径（成熟稳定，适用于高保真场景）

虚拟化技术通过抽象物理硬件，实现单台物理机承载成百上千个虚拟节点，是大规模靶场的核心基础。

1. 核心技术选型

   - **服务器虚拟化**：采用 **VMware vSphere、KVM、Xen** 等成熟方案，单台高性能物理机（如配备 2 颗 32 核 CPU、512GB 内存、10TB SSD）可虚拟出 **200-500 个轻量级虚拟机（VM）节点**；通过分布式集群（如 100 台物理机集群），可支撑 10 万 + VM 节点规模。
   - **网络虚拟化**：搭配 **Open vSwitch（OVS）、Cisco ACI** 等技术，构建虚拟交换机、路由器、防火墙，实现节点间的网络拓扑灵活编排（如星型、网状、层级架构），并支持流量隔离、带宽限制、延迟模拟。

2. 部署架构

   - 采用

     “管理节点 + 计算节点 + 存储节点” 的分布式集群架构：

     - 管理节点：负责资源调度（如基于 OpenStack 平台）、拓扑编排、任务下发；
     - 计算节点：承载虚拟靶标节点，通过共享存储（如 Ceph 分布式存储）实现镜像快速分发；
     - 存储节点：存储靶标镜像、漏洞环境、演练日志，保障高并发读写。

3. 优势与局限

   - 优势：**高保真度**，虚拟节点可完整模拟真实操作系统（如 Windows Server、Linux）、业务系统（如 Web 服务、数据库），支持复杂漏洞复现（如 CVE 漏洞利用）；
   - 局限：资源消耗较高，10 万 + 节点需投入大量物理服务器，成本相对昂贵。

### 基于**容器化 + 轻量级虚拟化**的搭建途径（高密度，适用于大规模并发场景）

针对 10 万 + 节点的高密度需求，容器化技术（Docker）+ 轻量级虚拟化（LXC、Firecracker）是更高效的选择，核心是**用更低资源占用实现更多节点部署**。

1. 核心技术选型
   - **容器化引擎**：采用 **Docker + Kubernetes（K8s）** 编排平台，单台物理机可部署 **1000-2000 个容器节点**，100 台物理机集群即可轻松支撑 10 万 + 节点规模；
   - **轻量级虚拟化增强**：对于需要内核隔离的场景，采用 **Firecracker（AWS 开源）、Kata Containers** 替代传统 VM，兼顾容器的轻量化和 VM 的强隔离性，单节点资源占用降低 50% 以上。
   - **靶标镜像标准化**：将 CVE 靶标、业务系统打包为**轻量化容器镜像**（如基于 Alpine Linux 的镜像体积仅几十 MB），通过 K8s 的镜像仓库（Harbor）实现快速分发和批量部署。
2. 关键优化策略
   - **资源超分技术**：在非高负载场景下，开启 CPU / 内存超分（如超分比 2:1），提升物理机资源利用率；
   - **动态扩缩容**：基于 K8s 的 HPA（Horizontal Pod Autoscaler），根据演练负载自动增减靶标节点数量；
   - **网络平面分离**：将管理网络、业务网络、监控网络分离，避免大规模节点通信拥堵。
3. 优势与局限
   - 优势：**高密度、低成本、快速部署**，适合大规模攻防演练、物联网节点模拟等场景；
   - 局限：容器共享宿主机内核，隔离性弱于 VM，部分需要内核级漏洞复现的场景适配性较差。

### 基于**网络仿真技术**的搭建途径（超高规模，适用于拓扑推演场景）

如果靶场核心需求是**模拟超大规模网络拓扑的行为特征**（如路由协议交互、流量传播规律），而非单个节点的高保真业务环境，可采用**网络仿真技术**，这是支撑**百万级节点**的最优方案。

1. 核心技术选型
   - **主流仿真平台**：**NS-3、OMNeT++、OPNET** 等， NS-3 尤其适合大规模网络仿真 —— 它通过**离散事件仿真**，无需为每个节点分配真实计算资源，而是通过数学模型模拟节点的网络行为（如数据包收发、协议处理）。
   - **混合仿真架构**：结合 “仿真节点 + 真实节点”，核心拓扑用 NS-3 仿真（支撑 10 万 + 虚拟节点），关键靶标节点（如攻击方、防御方核心设备）用真实 VM / 容器承载，兼顾规模与实战性。
2. 部署要点
   - 采用**分布式仿真**：将仿真任务拆分到多台物理机，通过 MPI（消息传递接口）实现节点间数据同步；
   - **模型简化优化**：对非核心节点采用简化模型（如仅模拟路由转发，不模拟应用层行为），降低计算负载。
3. 优势与局限
   - 优势：**极致规模**，单台高性能服务器可仿真 10 万 + 节点，成本极低；适合卫星网络、物联网等超大规模拓扑推演；
   - 局限：**低保真度**，无法模拟真实业务系统的漏洞和攻击效果，仅适用于网络行为研究。
