## 使用Git

### Git必要的配置

* 顶部打开一个新的terminal，运行：git --version

  * 如果输出版本表示已经识别到git
  * 如果没有输出版本，则在VScode的左下角点击⚙️->setting-> 搜索“git.path"
    * 在打开的setttings.json文件中填写git.path：`  "git.path": "C:\\Program Files\\Git\\cmd\\git.exe"`

* **配置Git用户**，在VSCode终端执行一下命令进行配置：

  ```
  git config --global user.name "JingtaoWang"
  git config --global user.email "wjt397618242@163.com"
  ```

  * 查看配置情况：`git config --global --list`

* **开启VSCode的git功能**

  * VSCode左侧找到**Source Control** 

    * 或者快捷键：Ctrl+Shift+G

    * 如果没有，去设置里确认：`Settings → 搜索 Git: Enabled` 确保勾选

  * 配置完git之后点击左侧诶对Source control会出现 Changes、Graph等

### Git 提交-Commit

* 打开自动保存：File -->AutoSave
* 代码修改后会在左侧Source control的change出现
* 如果要提交，输入message后commit & push
* 如果某一个文件的修改要撤回，直接这个文件右侧的`Discard Changes`后即可恢复此文件全部改动



### Git 拉取代码-Pull

* 终端直接执行：`git pull`

* 指定分支拉取: `git pull origin main`
* master分支拉取: `git pull origin master`
* 开发分支拉取: `git pull origin dev`

### 遇到的问题

* 问题1：git版本无法识别

  * git --version  执行报错：git : 无法将“git”项识别为 cmdlet、函数、脚本文件或可运行程序的名称。请检查名称的拼写，如果包括路径，请确保路径正确，然后再试 一次。

    * 原因：Git 已经安装（你说已安装），但没有加入系统 PATH，VS Code / PowerShell 找不到它。

    * 解决方案：手动添加git到环境变量

      * win+R -->sysdm.cpl-->高级-->环境变量

      * 在系统变量或者用户变量中找到`Path`,新增两行

        ```
        C:\Program Files\Git\bin
        C:\Program Files\Git\cmd
        ```

      * 保存并关闭所有窗口后重新打开VSCode

      * 再次执行git --version问题解决

## VScode 同时打开并运行多个工程

### 使用工作区Workspace

* 打开第一个工程：`File → Open Folder`
* 添加第二个工程：`File → Add Folder to Workspace`
* 保存工作区：`File → Save Workspace As`

**以之后双击这个workspace就会同时打开两个工程**

## VScode 如何连接远程服务器运行脚本

* 本地VScode安装插件：`Remote - SSH `

* 确认本地有SSH：`ssh -V`

* 测试目标服务器能否连接：

  * ssh root@172.16.151.41   /  非22端口：ssh -p 2222 root@192.168.1.100

* 配置SSH config

  * `ctrl+shift+P`后搜索`Remote-SSH: Open SSH Configuration File`

  * 选择`C:\Users\你的用户名\.ssh\config`

  * 写入以下文件：【多个文件可以】

    ```
    Host 41
        HostName 172.16.151.41
        User root
        Port 22
    ```

* 用VSCode连接远程服务器

  ```
  Ctrl + Shift + P
  → Remote-SSH: Connect to Host
  → 41
  ```

* 打开远程项目目录

  * 连接成功后：`File → Open Folder`并选择远程服务器上的项目目录

* 在远程服务器运行脚本

  * `Terminal → New Terminal`
  * cd 到指定路径后运行Python脚本

* 配置远程Python解释器

  * `Ctrl+Shift+P → Python: Select Interpreter` 选择服务器上的**Python**或者**Conda**

* 远程调试Python脚本

  * 在远程项目目录下创建：`.vscode/launch.json`

  * ```
    {
      "version": "0.2.0",
      "configurations": [
        {
          "name": "Remote Debug Python File",
          "type": "debugpy",
          "request": "launch",
          "program": "${file}",
          "console": "integratedTerminal",
          "justMyCode": true
        }
      ]
    }
    ```

    

# VScode for Vue

## 安装VSCode Extension

* 点击VScode左侧**Extension**
* 安装以下扩展：
  * Vue - Official
  * ESLint
  * Prettier - Code formatter

## 已有 Vue 项目 + pnpm + VS Code 运行

* 打开项目根目录：File --> Open Folder

* 打开VScode终端：顶部Terminal--> New Terminal

  * **确认pnpm已经安装：pnpm -v**【输出pnpm的版本号】
    * 如果提示没有pnpm，执行命令安装：npm install -g pnpm
  * 安装项目依赖【项目根目录下执行】：pnpm install 

* 打开`package.json`,搜索其中的`scripts`, 确认项目启动方式

  以leo_vue项目为例，找到的内容如下

  ```
    "scripts": {
      "serve": "vue-cli-service serve",
      "build": "vue-cli-service build",
      "lint": "vue-cli-service lint"
    },
  ```

  * 如果scripts下面的key包含了：“dev”：
    * terminal中执行命令运行项目：`pnpm dev`
  * 如果scripts下面的key包含`serve`[类似上面的示例]：
    * Terminal执行命令运行项目：`pnpm serve`

## 创建Vue项目

* 确认Node.js和npm可用

```
node -v
npm -v
```

### npm创建Vue项目

* 打开VSCode终端：`Terminal-->New Terminal`
* 创建项目，根据提示选择项目配置：`npm create vue@latest`
* 终端cd进入项目：`cd 项目名`
* 安装依赖：`npm install`
* 启动项目:`npm run dev`或 `npm run serve`

### pnpm创建Vue项目

* 安装pnpm：`npm install -g pnpm`
  * 确认安装完成：`pnpm -v`
* 创建项目：`pnpm create vue@latest`
* 终端cd进入项目：`cd 项目名`
* 安装依赖：`npm install`
* 启动项目:`npm run dev`或 `npm run serve`

### 项目创建后用VScode打开

* terminal cd 到项目根目录后执行：`code .`
* File ->Open Folder ->打开根目录



# VScode for Python

## 环境变量配置及插件安装

* 插件安装`Python、Pylance`：发布者一般为Microsoft，且自动安装
* 选择Python解释器：

## Python解释器选择

* `Ctrl+shift+P`后搜索：`Python: Select Interpreter`
  * 可根据实际情况进行选择

## 运行Python文件

* 直接点击右上角进行运行
* 终端运行：
  * cd到指定目录后执行：`python3 .\main.py`

## 安装依赖pip

* **如果有requirements.txt**: 终端执行:`pip install -r requirements.txt`

* 单独安装某个包[类似linux进行]：`pip3 install XXXX`
* 如果pip不识别：
  * `python -m pip install -r requirements.txt`

## 配置虚拟环境&conda环境 

* 确认conda环境可用：`conda --version`

  * 若未识别到具体情况则需将conda加入环境变量

* 安装好conda环境后，创建conda环境等等步骤类似linux

* 选择conda环境作为解释器

  * `ctrl+shift+p`：输入 `Python:Select Interpreter`后选择类似`Python 3.12.x ('py312': conda)`

  







# VScode for Java 

## 确认并配置windows中的Maven环境变量

* 确认系统环境变量中有没有Maven：`where mvn`
  * 没有输出则说明环境变量中没有Maven

* 使用IJ中的maven路径：`Settings→Preferences→Build,Execution,Deployment→Build Tools→Maven→Maven home path`
* 将IJ自带的maven路径加入环境变量
  * D:\ij\IntelliJ IDEA 2025.1\plugins\maven\lib\maven3\bin

## 安装VScode extension

扩展商店中安装：

* `Extension Pack for Java`: 包含Java语言支持、调试器、Maven支持、测试运行器等
* `Spring Boot Extension Pack`:包含Springboot Tools、Spring Initializr



## 打开并运行Springboot项目工程

* 打开包含`pom.xml`的目录
* 下载依赖：`mvn clean install`
